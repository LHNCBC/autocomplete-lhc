autocomplete-lhc

This is a package of JavaScript autocompleters based on PrototypeJS and Scriptaculous. These were originally developed to meet the requirements of the NLM PHR project. Changes from the Scriptaculous autocompleters:

  1. Supports coded-value lists
  2. Provides a way to get extra information about the record associated with a list item. (See recordDataRequester.js.)
  3. Numbered list items (optional).
  4. Two-column lists when there is not enough space on the page to show the list in a single column.
  5. Partial support for multi-select lists. (Multi-select lists do not currently support entry of text not on the list.)
  6. Support for list suggestions in the search (AJAX) autocompleter, when the user does not pick an item in the list.
  7. Can require that an entry in the field match the list.
  8. Event listeners for various things like "list selection".
  9. "autofill" option -- fills in the field automatically if there is just one list item
  10. Heading items in lists
  11. Default values (optional), which gets placed in the field when the field gets focus (along with showing the list).
  12. A "see more" link on the search autocompleter which provides an expanded list of items.
  13. Search buttons (optional) for search autocompleters to make them show an expanded results list. (Without a button, the user can click on the "see more" link.)
  14. A results cache for the search autocompleter, so that repeats of AJAX calls for the same list items are not necessary.
